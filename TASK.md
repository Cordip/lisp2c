## **Lisp-машина** 

Разработайте среду исполнения (runtime) Lisp. Трансляция должна осуществляться в код на  языке C, который должен компилироваться GCC без всяких изменений.  

Реализация должна содержать минимальный набор элементов, которые не могут быть  выражены на самом Lisp, все остальные необходимые элементы языка должны быть  выражены в этом минимально базисе (определение функции, аппликация, определение  списка, и ряд других). Может быть реализовано подмножество любого диалекта Lisp (можете  придумать свой). Необходимые элементы (не все они обязаны входить в базис): 

    * Определение функции 
    * Определение статического лексического контекста (let) 
    * Рекурсия 
    * Доступ к вызовам платформы (кроме native assembler) 
    * Ветвление, логические связки с отложенным порядком исполнения 
    * Хвостовая рекурсия 
    * Макрос 
    * Ввод-вывод
    * Механизм, подобный макросу, но генерирующий не Lisp-код, а непосредственно код  для платформы исполнения, т.е. C 

## **Общие требования к выполнению заданий** 

1. Все задания рассчитаны на двух человек, однако допускается выполнение в одиночку. 
2. Большинство заданий рамочные и требуют детализации требований. Это необходимо сделать  в форме документа в первые 2 недели выполнения проекта. При выполнении вдвоем также  требуется разделить зоны ответственности в рамках задачи.  
3. Допускается творчески переработать задание или придумать свое задание на разработку  фреймворка по аналогии с предлагаемыми. В этом случае требуется согласование с  преподавателем в первую неделю.  
4. Для всех заданий требуется покрытие тестами. 
5. Для всех заданий требуется дополнительный код, демонстрирующий использование  разработанного фреймворка.  
6. Если не указано обратное, то **целевым** (т.е. для чего фреймворк предназначен) языком  является Java. Язык/фреймворк реализации в каждом случае свой и может обсуждаться.
